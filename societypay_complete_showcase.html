SocietyPay
// AI-Powered Society Management  ·  Full-Stack Mobile Platform  ·  NIT Silchar
React NativeTypeScriptNode.jsMongoDBPythonFastAPIRazorpayScikit-learnNaive Bayes
A full-stack mobile platform that brings AI to residential society management — automating maintenance payments, predicting defaulters with ML risk scoring, and intelligently routing complaints using NLP classifiers.
The Problem
Maintenance payments are delayed and hard to track across flats
Complaints pile up with no priority, category or routing logic
Secretaries have zero visibility into who is likely to default
Communication between residents and management is scattered

SecuritySS — Smart Visitor Management System
Smart Security Visitor System
// AI-Powered Visitor Tracking & Anomaly Detection  ·  Real-Time ML  ·  NIT Silchar
React NativeTypeScriptNode.jsMongoDBPythonFlask / FastAPIIsolation Forestchart-kit
A production-level mobile security dashboard combining real-time visitor tracking with unsupervised ML anomaly detection — guards see live behavior flags, weekly trend charts, full visit history and instant risk scoring, all auto-refreshing every 10 minutes.
The Problem
Traditional systems do not track visitor behavior over time
Suspicious patterns go undetected without ML analysis
No real-time monitoring — guards rely on manual registers
Zero analytics or insights for security decision-making
The Solution
Track visitor entries and exits with automatic timestamps
Detect suspicious behavior using Isolation Forest ML model
Visualize weekly visitor trends with real-time graph charts
Auto-refresh every 10 minutes with live anomaly re-scoring

Challenges Faced & Solutions
ChallengeChart crashing with NaN values
Data sanitization layer added — missing days fill with 0, stable Mon→Sun ordering enforced, NaN-safe rendering throughout the graph pipeline
ChallengeReal-time behavior tracking
Solved with interval-based API refresh every 10 minutes — backend re-scores all active visitors and pushes updated anomalyRisk to frontend
ChallengeML data quality for anomaly detection
Improved through feature engineering (visit frequency, hour, duration, unique flats) and data normalization before Isolation Forest inference
